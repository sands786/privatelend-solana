How Arcium MXE protects your position
All sensitive lending computations run inside Arcium's Multi-party computation Execution Environment. The Solana program stores only encrypted ciphertexts — raw collateral, LTV values and health factors are never visible on-chain.
// step 01
Encrypt
Collateral and borrow amounts are encrypted client-side before submission. Solana stores only ciphertexts — no plaintext values touch the chain.
// step 02
Compute in MXE
LTV ratios, health factors and interest rates computed inside Arcium MXE using multi-party computation. No single node sees raw values.
// step 03
Private Liquidation
Liquidation only triggers when MXE issues a ZK proof that health < 1.0. The threshold is never exposed — bots cannot front-run or predict it.
